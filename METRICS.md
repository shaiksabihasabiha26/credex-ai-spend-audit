# Metrics Strategy

# North Star Metric

## Completed Audit Reports Per Week

This is the most important metric because:
- it measures actual product usage
- users only complete audits if they perceive value
- completed audits directly feed lead generation

Traffic alone is not meaningful if users never finish the audit flow.

---

# Input Metrics

## 1. Audit Completion Rate

Percentage of users who finish the audit after landing on the page.

This measures:
- onboarding clarity
- UX quality
- user intent alignment

---

## 2. Lead Capture Rate

Percentage of completed audits that submit email/contact information.

This measures:
- trust
- perceived audit value
- conversion quality

---

## 3. Share Rate

Percentage of users who share audit results publicly.

This measures:
- virality
- screenshot quality
- distribution efficiency

---

# What I Would Instrument First

If scaling the product, I would track:

- landing page conversion funnels
- audit dropoff points
- most common AI tools entered
- average projected savings
- consultation request rates
- shareable link usage
- repeat audit behavior

Tools:
- PostHog
- Google Analytics
- Vercel Analytics

---

# Pivot Threshold

If fewer than:
- 10% of visitors complete audits
OR
- 5% of audit users submit lead information

after meaningful traffic volume, I would reconsider:
- onboarding UX
- recommendation quality
- positioning
- target audience

---

# Long-Term Metrics

As the platform grows, additional metrics would include:
- customer acquisition cost
- consultation-to-sale conversion
- average customer value
- recurring audit usage
- infrastructure credit purchases driven by the tool
